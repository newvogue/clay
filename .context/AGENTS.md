# Правила для AI-агента (Clay-specific)

> Этот файл читается агентом при старте в проекте Clay.
> Глобальные правила стиля общения — в `~/.opencode/AGENTS.md`.

## Перед началом работы прочитай

1. `.context/README.md` — как устроена память
2. `.context/state.md` — текущее состояние
3. `.context/handoffs/current.md` — последнее задание

## После работы обнови

1. `.context/reports/last.md` — что сделал
2. `.context/state.md` — что изменилось
3. Опционально: `.context/observations/YYYY-MM/obs-NNN.md` — заметки

## Конвенции

- Не пиши бинарные файлы в `.context/`
- Коммить `.context/` в git
- Используй формат записей: Контекст → Решение → Why / How to apply
- Ссылайся на ID: `#obs-2026-06-01-001`, `#decision-0001`
- Privacy: тег `<private>` в начале body исключает из индекса

Стиль общения с пользователем (русский, кратко, эмодзи, пошаговое подтверждение) — см. `~/.opencode/AGENTS.md` и `~/.opencode/memory/communication-rules.md`.

## 🚀 Стратегия запуска (экономия контекста)

**Правило Emma** (применяется ко всем сессиям в проектах с `.context/`):

При старте новой сессии, **если нет явного указания** на написание/редактирование кода (edit, write, cp, mv, rm, bash-команды с side-effects), **запустить `explore` subagent** для чтения и исследования файлов из `.context/`. Возвращённый summary (~1-2K токенов) — основной контекст. Это экономит **~40K токенов** (~20% контекстного окна), которые иначе ушли бы на `read` 4-5 файлов в основном контексте.

**Когда применять:**
- Recon (прочитать state.md, handoffs/current.md, observations, recon-отчёт)
- "Покажи где у нас X" / "Что мы решили про Y" / review-вопросы
- Plan-фаза перед gated-implementation
- Любой read-only запрос

**Когда НЕ применять (делать самому в основном контексте):**
- Прямое code action (edit/write/cp/mv/rm)
- Простой Q&A (короткий ответ в чат)
- Smoke / quick check (быстрый ответ нужен)
- Действие требует chain-реакции в основном контексте (например, code action сразу после recon)

**Промпт для `explore`:** чёткий и ограниченный — какие файлы прочитать, что вернуть. Например:

```python
task(
    description="Read .context files and return summary",
    prompt="Прочитай эти файлы и верни КРАТКОЕ summary:\n1. .context/state.md — точка остановки\n2. .context/handoffs/current.md — задание\n3. .context/observations/2026-06/obs-2026-06-01-003 — критичное observation\n\nВерни в формате:\n- Текущая точка: <1 строка>\n- Задание: <1-2 строки>\n- Критичные constraints: <bullet list>\n- Что НЕ делать: <bullet list>",
    subagent_type="explore",
)
```

Подробнее про subagent'ов и их контекстную изоляцию — в `~/.opencode/AGENTS.md`.

## 🔄 Session Handoff Protocol (Clay-specific)

**Триггеры** — когда Emma пишет одно из:
- "переходим к новой сессии"
- "завершаем сессию и переходим к новой"
- "готово, переход" / "session handoff"
- "стоп сессия" / "новая сессия"

**Действия агента:**

1. **Спросить** "Сделаем точку остановки?" (если Emma не в `да` режиме)
2. **После подтверждения** — обновить 3 файла:
   - `.context/state.md` — что в работе, что дальше
   - `.context/reports/last.md` — что сделано в этой сессии
   - (опционально) `.context/observations/YYYY-MM/obs-NNN.md` — находка, если есть
3. **Сгенерировать промпт** по шаблону из `.context/handoff-template.md` (подставить `state_summary`, `next_step`, `pending_question`, `extra_files_block`)
4. **Выдать промпт Emma** в чат (≤15 строк, готов к копированию)
5. **Напомнить** 3 шага Emma: копировать → закрыть → открыть → вставить

**Чего НЕ делать:**
- ❌ Не глобально (правило работает **только** в проектах с `.context/AGENTS.md`)
- ❌ Не выдавать промпт без обновления `state.md` (агент в новой сессии увидит устаревшее)
- ❌ Не включать в промпт ссылки на snapshot для архитектора (это не для агента)
- ❌ Не выдумывать "следующий шаг" если его нет в `state.md` — лучше спросить Emma

**Где это правило:** здесь, в `AGENTS.md` (Clay-specific). В глобальном `~/.opencode/AGENTS.md` такого правила **нет** — это намеренно.

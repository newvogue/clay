### Архитектура (ratified)

1. **Async job, не sync/threadpool.** `IngestionCycleJob` регистрируется как coroutine на `AsyncIOScheduler` напрямую (без `ThreadPoolExecutor`). Зеркалит `POST /ingestion/run` 1:1 (тот же async-путь, та же sync-DB на loop).
2. **Concurrency-guard = `asyncio.Lock` + `is_running`, service-level.** Lock **покрывает весь `run_once`** (market + context + commit — TOCTOU mitigation explicit). Manual route → **409 Conflict** (operator feedback). Scheduler-job → **skip + log** (quiet; `max_instances=1` + `coalesce=True` страхуют).
3. **Flag-gating:** `CLAY_SCHEDULER_INGESTION_ENABLED: bool = True` (независимо от `scheduler.enabled`, по аналогии с `reliability_enabled`). Интервал — `ingestion_cycle_interval_seconds=60` (уже в `SchedulerSettings:32, 59`).
4. **emit-семантика:** `emit=False` (scheduler) скипает **только audit+event** (observability), DB writes — mandatory. Manual route `emit=True`. **Anti-flood signal = transition** — `(incidents_present bool, freshness_state_transitions int)` tuple. INSERT-based emit = флуд (klines меняются каждый тик), поэтому **НЕ** primary signal. Steady data-flow → no emit. **Freshness transition** детектится через `upsert_freshness_status → bool` refactor (см. fragment B' ниже) — только реальная смена `freshness_state` инкрементит счётчик, no-op timestamp-only touch = не transition.
5. **Error-policy:** ДВА wrapper'а, общий error-хелпер. **Sync `_run_safely` остаётся sync** (B3b/B4 health/reliability — registered sync → ThreadPoolExecutor, **ЗОЛОЕ неизменное**). **Новый async `_arun_safely`** для B5 (registered coroutine → event loop, mirror route). Оба зовут общий `_handle_job_error(exc, pre_status, on_error)` — логика error-policy в одном месте, дублирования нет, routing корректный.
6. **DI:** явный конструктор (`session_factory`, `ingestion_service`-compatible, `audit_writer`, `event_bus`). **NO** `import clay.bootstrap` (A6 invariant, grep=0).
7. **Counter split** `market_records_written` → `inserted` + `updated` (механическое изменение, low-cost). Existing field становится computed property `= inserted + updated` для backward-compat. New fields в `IngestionRunSummary` + `as_payload()`.

**A. `IngestionCycleService` — lock + is_running + emit-gating (ingestion/service.py):**

```python
class IngestionCycleBusy(RuntimeError):
    """Raised when a cycle is already in progress."""


class IngestionCycleService:
    def __init__(
        self,
        settings: IngestionSettings,
        market_service: MarketIngestionService,
        context_manager: ContextConnectorManager,
        audit_writer: AuditWriter | None = None,
        event_bus: EventBus | None = None,
    ) -> None:
        self.settings = settings
        self.market_service = market_service
        self.context_manager = context_manager
        self._audit_writer = audit_writer
        self._event_bus = event_bus
        self._lock = asyncio.Lock()

    @property
    def is_running(self) -> bool:
        return self._lock.locked()

    async def run_once(self, session: Session, *, emit: bool = True) -> IngestionRunSummary:
        if self._lock.locked():
            raise IngestionCycleBusy("ingestion cycle already running")
        async with self._lock:
            summary = await self._do_run_once(session)
        if emit:
            self.emit_cycle_events(summary)
        return summary

    async def _do_run_once(self, session: Session) -> IngestionRunSummary:
        # ... existing pipeline (market → context → commit) ...
        # Update call sites for upsert_market_bars tuple return
        return summary

    def emit_cycle_events(self, summary: IngestionRunSummary) -> None:
        if self._audit_writer is None or self._event_bus is None:
            return  # нечего эмитить (test/incomplete wiring)
        payload = summary.as_payload()
        self._audit_writer.write("ingestion.run", payload)
        self._event_bus.publish("ingestion.updated", {"event_type": "ingestion.run", **payload})
```

**Note:** emit happens **после** lock release (не под локом — observability не нужно сериализовать). `is_running` property is a fast non-blocking check via `lock.locked()`.

**B'. `IngestionCycleJob` — async, B4 pattern (scheduler/jobs.py):**

```python
class _IngestionCycleRunnable(Protocol):
    async def run_once(self, session: Session, *, emit: bool = ...) -> IngestionRunSummary: ...
    def emit_cycle_events(self, summary: IngestionRunSummary) -> None: ...


class IngestionCycleJob:
    def __init__(
        self,
        ingestion_service: _IngestionCycleRunnable,
        session_factory: sessionmaker,
        audit_writer: AuditWriter,
        event_bus: EventBus,  # reserved (v2: ingestion.tick)
    ) -> None:
        self._service = ingestion_service
        self._session_factory = session_factory
        self._audit_writer = audit_writer
        self._event_bus = event_bus
        self._cache: tuple[bool, int] | None = None  # (incidents_present, freshness_state_transitions)
        self._failing: bool = False  # B4 #11 anti-flood

    async def run(self) -> None:  # ← ASYNC coroutine
        if self._service.is_running:
            logger.info("clay.scheduler: ingestion cycle already running, skip tick")
            return
        with self._session_factory() as session:
            try:
                summary = await self._service.run_once(session, emit=False)
            except IngestionCycleBusy:
                # Race: started between is_running check and run_once
                logger.info("clay.scheduler: ingestion cycle started mid-tick, skip")
                return
            session.commit()
        # B4 #11: success closes the failing episode
        self._failing = False
        new_state = (bool(summary.incidents), summary.freshness_state_transitions)
        if self._cache is None:
            self._cache = new_state
            return
        if new_state == self._cache:
            return
        # Transition: emit through the public method
        self._service.emit_cycle_events(summary)
        self._cache = new_state

    def on_error(self, exc: Exception) -> None:
        if not self._failing:
            self._audit_writer.write("ingestion.cycle_failed", {"error": str(exc)})
        self._failing = True
        logger.exception(
            "clay.scheduler: ingestion cycle failed; "
            "session-scheduler NOT marked ERROR (isolated policy)",
        )
        # НЕ re-raise, НЕ mutate registry — session-scheduler НЕ трогаем
```

**Note:** `freshness_state_transitions` (not `freshness_updates_written`) — transition signal. `upsert_freshness_status` returns `bool` — `True` только при реальной смене `freshness_state` (INSERT или UPDATE с разным значением). Timestamp-only touch (steady) = `False` → counter не растёт → no emit. (See [repositories_market.py:58-91] original behavior — unconditional UPDATE; refactor per Emma's Поправка 2.)

**C'. `ClayScheduler` — TWO wrappers, shared error-policy (scheduler/service.py):**

```python
def _run_safely(
    self,
    job_callable: Callable[[], None],
    *,
    on_error: Callable[[Exception], None] | None = None,
) -> None:
    """Sync wrapper — REGISTERED AS SYNC with APScheduler → ThreadPoolExecutor.
    
    Used by HealthTickJob (B3b) and ReliabilityRecheckJob (B4). Their
    sync-DB-I/O MUST stay in threadpool (ADR-007, B0 §11.1 mitigation).
    APScheduler routing is determined by the registered callable's
    signature (sync vs async def), NOT by what it calls internally.
    
    ZERO behavioral changes from B3b/B4. Routing preserved.
    """
    pre_status = self._registry.get(self._SERVICE_ID).status
    try:
        job_callable()
    except Exception as exc:
        self._handle_job_error(exc, pre_status, on_error)


async def _arun_safely(
    self,
    job_callable: Callable[[], Awaitable[None]],
    *,
    on_error: Callable[[Exception], None] | None = None,
) -> None:
    """Async wrapper — REGISTERED AS COROUTINE with APScheduler → event loop.
    
    Used by IngestionCycleJob (B5). Mirrors POST /ingestion/run async
    path 1:1 (same sync-DB on loop as the route does in production today,
    per Emma's [MED-A] ratification). Per-scheduler-job wrapper signature
    controls APScheduler routing.
    """
    pre_status = self._registry.get(self._SERVICE_ID).status
    try:
        await job_callable()
    except Exception as exc:
        self._handle_job_error(exc, pre_status, on_error)


def _handle_job_error(
    self,
    exc: Exception,
    pre_status: ServiceStatus,
    on_error: Callable[[Exception], None] | None,
) -> None:
    """Shared error-policy between _run_safely (sync) and _arun_safely (async).
    
    - Custom on_error (B4 pattern, e.g. ReliabilityRecheckJob.on_error):
      delegated, no registry mutation, no re-raise.
    - Default (B3b pattern, e.g. HealthTickJob): session-scheduler → ERROR
      + audit on transition + log.exception, no re-raise.
    """
    if on_error is not None:
        on_error(exc)
        return
    # Default B3b path
    self._registry.update_status(self._SERVICE_ID, ServiceStatus.ERROR, error=str(exc))
    if pre_status != ServiceStatus.ERROR:
        self._audit_writer.write("service.status_changed", {
            "service_id": self._SERVICE_ID,
            "from": pre_status.value, "to": ServiceStatus.ERROR.value, "error": str(exc),
        })
    logger.exception("clay.scheduler: job raised; marking session-scheduler ERROR")
```

**Routing matrix:**

| Job | Wrapper registered with APScheduler | APScheduler sees | Runs in | Rationale |
|---|---|---|---|---|
| HealthTickJob (B3b) | `self._run_safely` (sync) | sync callable | ThreadPoolExecutor (`executor="default"`) | sync DB-I/O не блокирует loop |
| ReliabilityRecheckJob (B4) | `self._run_safely` (sync) | sync callable | ThreadPoolExecutor | sync DB-I/O не блокирует loop |
| **IngestionCycleJob (B5)** | `self._arun_safely` (async) | coroutine function | event loop | mirror `POST /ingestion/run` 1:1, sync-DB-on-loop = identical to route (works in prod today) |

**Why two wrappers, not one async `inspect.isawaitable`:** the registered callable's signature (`sync def` vs `async def`) determines APScheduler's routing, NOT what the wrapper calls internally. Converting `_run_safely` to `async def` would silently move B3b/B4 sync-DB onto the event loop (regression). Two wrappers + shared error-helper = correct routing + no duplication.

### B''. `upsert_freshness_status` refactor (repositories_market.py:58-91, per Emma's Поправка 2):

```python
def upsert_freshness_status(
    self,
    *,
    symbol: str,
    timeframe: str,
    freshness_state: str,
    evaluated_at: datetime,
    latest_bar_open_time: datetime | None,
    is_stale: bool,
) -> bool:
    """Returns True if a freshness state transition occurred, False otherwise.
    
    - INSERT (no existing row): True (first observation = transition from "unknown")
    - UPDATE with same state: False (timestamp-only touch, NO transition)
    - UPDATE with different state: True (real state change)
    """
    existing = self.session.scalar(
        select(MarketFreshnessStatus).where(
            MarketFreshnessStatus.symbol == symbol,
            MarketFreshnessStatus.timeframe == timeframe,
        ),
    )
    if existing is None:
        self.session.add(
            MarketFreshnessStatus(
                symbol=symbol,
                timeframe=timeframe,
                freshness_state=freshness_state,
                evaluated_at=evaluated_at,
                latest_bar_open_time=latest_bar_open_time,
                is_stale=is_stale,
            ),
        )
        return True  # INSERT = transition

    state_changed = existing.freshness_state != freshness_state
    existing.freshness_state = freshness_state
    existing.evaluated_at = evaluated_at
    existing.latest_bar_open_time = latest_bar_open_time
    existing.is_stale = is_stale
    return state_changed
```

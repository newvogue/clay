**Паттерны (для новых persistence-таблиц в A1):**
- `class Xxx(Base)` + `__tablename__` + `__table_args__ = {"schema": "ops"}`
- Типы: `Mapped[int]` / `Mapped[str]` / `Mapped[datetime]` / `Mapped[X | None]`
- `mapped_column(primary_key=True)` / `mapped_column(String(N), index=True)` / `mapped_column(DateTime(timezone=True))`
- `nullable=True` для опциональных, `nullable=False` (по умолчанию) для обязательных
- `default="..."` для Python-side default, `server_default="..."` для SQL-side (использовать в миграции, не в модели)
- `Text()` для длинного текста без ограничения

## §3. `SQLITE_SCHEMA_TRANSLATE_MAP`

**Файл:** `src/clay/db/session.py:7`

```python
from sqlalchemy import Engine, create_engine
from sqlalchemy.orm import sessionmaker

from clay.settings.ingestion import IngestionSettings


SQLITE_SCHEMA_TRANSLATE_MAP = {
    "market": None,
    "context": None,
    "ops": None,
    "demo": None,
    "review": None,
    "knowledge": None,
    "validation": None,
}


def build_engine(settings: IngestionSettings | None = None) -> Engine:
    resolved = settings or IngestionSettings()
    engine_kwargs: dict[str, object] = {"future": True}

    if resolved.database_url.startswith("sqlite"):
        engine_kwargs["connect_args"] = {"check_same_thread": False}
        engine_kwargs["execution_options"] = {
            "schema_translate_map": SQLITE_SCHEMA_TRANSLATE_MAP,
        }

    return create_engine(resolved.database_url, **engine_kwargs)


def build_session_factory(
    settings: IngestionSettings | None = None,
) -> sessionmaker:
    return sessionmaker(
        bind=build_engine(settings),
        autoflush=False,
        autocommit=False,
        expire_on_commit=False,
    )
```

**Что важно для A1:**
- Если новая таблица в `ops` schema — она автоматически попадёт в TRANSLATE_MAP (там все 7 schema мапятся в `None` для SQLite). **Ничего не нужно делать.**
- `build_engine` + `build_session_factory` — стандартная фабрика SQLAlchemy 2.0.
- `IngestionSettings` (env-prefix `CLAY_`) — для конфига БД в тестах.
- `create_engine` только с `database_url`, без `schema=` параметра (т.к. schema определяется в `__table_args__` моделей).

## §8. Сюрпризы и уточнения для архитектора

1. **Файл `src/clay/db/schemas/ops.py` НЕ существует** — schema идёт через `db/models_ops.py` (одна общая declarative base). Архитектор оперировал именем из старого intelligence.

2. **`roles` и `models` — это registry, не runtime-state** — они не меняются в runtime, инициализируются один раз в `__init__` из `_build_*_registry()`. Для persistence **можно НЕ персистить** (только assignments).

3. **`assignments: dict[str, str]` — это и есть runtime-state E5** — маппинг role_id → model_id. Это то, что меняется через `apply_assignment()`.

4. **В `_last_reviewed_at` нет audit-trail** — хранится только timestamp последнего review, не история reviews. Audit пишется в `audit.jsonl` отдельно.

5. **`_active_session` имеет `paused_at`** — `datetime | None`, default `None`. При persist нужно учесть nullable.

6. **`_strategy_mode` в validation_lab — простая строка** (default `"momentum"`), не enum. Persistence может быть строковым полем с server_default.

7. **Workspace `_focus_*` — это runtime-указатель** (что сейчас выбрано), а не полная история focus-изменений. Audit пишется через `event_bus` отдельно.

8. **Reliability `_last_rechecked_at` — единственная timestamp-точка** для E12. Audit `reliability.rechecked` пишется в JSONL.

9. **5 dataclass'ов всего** в in-memory state (для reference при проектировании DDL):
   - `RoleDefinition`, `ModelVersion` (frozen) — registry
   - `PendingReview` — review card
   - `ActiveSessionRecord`, `PendingReplacementReview` — session lifecycle
   - `PairContext` (slots=True) — workspace internal, не state

10. **`del session` на строке 90** в `ai_control/service.py:90` явно маркирует E5 как in-memory. При persist — параметр `session` начнёт использоваться.

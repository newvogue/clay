# Контекст проекта

Эта папка — долгосрочная память проекта. Vendor-agnostic, читается любым агентом и человеком.

## Как начать работу (для нового агента)

1. Прочитай `INDEX.md` — указатель всех файлов
2. Прочитай `state.md` — текущее состояние
3. Прочитай `handoffs/current.md` — последнее задание
4. Сделай работу
5. Обнови `reports/last.md` и `state.md`

## Главное

- **Никогда не удаляй `.context/` без бэкапа** — это единственный источник правды
- **Всё в markdown**, никаких бинарных файлов
- **Коммить `.context/` в git** — это история проекта
- **Горячее в `state.md`**, детали в `sessions/` и `observations/`
- **Решения в `decisions/`** с обоснованием (почему именно так)
- **Privacy**: тег `<private>` в начале body скрывает запись из `INDEX.md`

## Структура

- `state.md` — текущее состояние (горячее, ≤25KB)
- `roadmap.md` — план: что сделано / в работе / дальше
- `architecture.md` — схема проекта
- `memory/` — долгосрочные знания (по типам)
- `decisions/` — ADR (Architecture Decision Records)
- `handoffs/` — входящие от архитектора
- `reports/` — исходящие от агента
- `observations/` — авто-заметки по датам
- `skills/` — ленивая загрузка контекста

## Бэкап и передача

```bash
# Снапшот
tar czf context-$(date +%F).tar.gz .context/

# Передать другому агенту
scp context-2026-06-01.tar.gz new-agent@host:/path/to/project/

# Восстановить
tar xzf context-2026-06-01.tar.gz
```

Никакой зависимости от конкретного AI-инструмента. Только markdown и git.
